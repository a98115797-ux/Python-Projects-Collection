Section 05

Key Concepts

Tokenizing

Turns "2 + 3" into [NUMBER 2, OP +, NUMBER 3].

Shunting-yard

Reorders infix tokens into RPN, respecting precedence (** > * / % > + -), associativity (** is right-associative), parentheses, and unary minus.

RPN evaluation

A stack: push numbers, pop operands when you hit an operator.

Why no eval?

eval(user_input) can run arbitrary code. A parser is safe by construction.

"2 + 3 * 4"
  -> tokenize  -> [2] [+] [3] [*] [4]
  -> shunting-yard -> RPN: 2 3 4 * +
  -> eval on a stack -> 14.0

Section 08

Complete Source Code

The engine's public API lives in models.py; the GUI in gui.py; entry point in main.py; settings and logging in config.py; tests in tests.py. Core of models.py:

models.py (core)
class CalculatorError(Exception):
    """Raised when an expression cannot be parsed or evaluated."""


_PRECEDENCE = {
    "u-": (4, True),   # unary minus
    "**": (3, True),
    "*": (2, False), "/": (2, False), "%": (2, False),
    "+": (1, False), "-": (1, False),
}


def eval_rpn(rpn):
    """Evaluate a Reverse Polish Notation token list."""
    stack = []
    for token in rpn:
        if token.kind == "NUMBER":
            stack.append(float(token.value))
            continue
        op = token.value
        if op == "u-":
            stack.append(-stack.pop())
        else:
            right = stack.pop()
            left = stack.pop()
            stack.append(_apply(op, right, left))
    if len(stack) != 1:
        raise CalculatorError("Invalid expression.")
    return stack[0]


class Calculator:
    """A safe arithmetic calculator with an in-memory history."""

    def evaluate(self, expression: str) -> float:
        if not expression or not expression.strip():
            raise CalculatorError("Expression is empty.")
        tokens = tokenize(expression)
        rpn = to_rpn(tokens)
        result = eval_rpn(rpn)
        self._history.append(HistoryEntry(expression.strip(), result))
        return result

The full tokenizer, shunting-yard to_rpn, GUI, and tests are in the repository files next to this guide.

Section 09

Code Walkthrough

config.py centralizes app metadata and sets up logging. models.py is the pure engine: Token (a frozen dataclass), tokenize (a lexer), to_rpn (shunting-yard, detecting unary minus from the previous token), eval_rpn (stack machine), and the Calculator service holding history via HistoryEntry. gui.py is a thin Tkinter layer that only builds widgets and calls Calculator.evaluate. main.py parses args and dispatches to REPL, GUI, or one-shot evaluation. The engine never imports the UI, so it stays testable in isolation.

Section 14

Deep Dive: precedence and associativity

The _PRECEDENCE table encodes both a level and a right-associativity flag. The shunting-yard loop pops higher-or-equal-precedence operators before pushing the current one, except for right-associative operators (**, unary minus), where it stops at strictly-higher precedence. That single rule is why 2 ** 3 ** 2 groups as 2 ** (3 ** 2). Extend the table with functions or constants to grow the language.
