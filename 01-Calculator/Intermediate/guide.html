Section 05

Key Concepts

Modules and imports

main.py imports helpers from utils.py. Keeping the menu separate from the plumbing makes each file easier to read and test.

Raising exceptions

Instead of returning an error string, compute() raises ZeroDivisionError, and the caller decides how to react.

JSON persistence

We serialize a list of dicts with json.dump and read it back with json.load, so history survives between runs.

[ Menu ]
  |-- 1 New       -> read + compute -> save to history.json -> back
  |-- 2 History   -> load + print -> back
  |-- 3 Clear     -> blank lines -> back
  '-- 4 Quit      -> exit

Complete Source Code

utils.py
"""Helper functions for the Intermediate calculator."""

import json
import os
from datetime import datetime

DATA_DIR = os.path.join(os.path.dirname(__file__), "data")
HISTORY_FILE = os.path.join(DATA_DIR, "history.json")
OPERATORS = {"+", "-", "*", "/"}


def read_number(prompt):
    """Prompt until the user enters a valid number."""
    while True:
        raw = input(prompt).strip()
        try:
            return float(raw)
        except ValueError:
            print("  Invalid number. Try again (example: 3.14).")


def compute(a, op, b):
    """Apply op to a and b. Raises ZeroDivisionError / ValueError."""
    if op == "+":
        return a + b
    if op == "-":
        return a - b
    if op == "*":
        return a * b
    if op == "/":
        if b == 0:
            raise ZeroDivisionError("Division by zero is undefined.")
        return a / b
    raise ValueError(f"Unknown operator: {op!r}")


def save_entry(expression, result):
    """Append one calculation to the history file."""
    os.makedirs(DATA_DIR, exist_ok=True)
    history = load_history()
    history.append({
        "expression": expression,
        "result": result,
        "timestamp": datetime.now().isoformat(timespec="seconds"),
    })
    with open(HISTORY_FILE, "w", encoding="utf-8") as file:
        json.dump(history, file, indent=2)
main.py
"""Intermediate Calculator: menu-driven console app with saved history."""

from utils import compute, load_history, read_number, read_operator, save_entry


def do_calculation():
    a = read_number("First number:  ")
    op = read_operator("Operator (+ - * /): ")
    b = read_number("Second number: ")
    try:
        result = compute(a, op, b)
    except ZeroDivisionError as error:
        print(f"  {error}")
        return
    print(f"  = {result}")
    save_entry(f"{a} {op} {b}", result)


def main():
    while True:
        choice = input("Option (1 new, 2 history, 4 quit): ").strip()
        if choice == "1":
            do_calculation()
        elif choice == "4":
            break

Section 09

Code Walkthrough

utils.py owns validation, maths (compute, which now raises instead of returning strings), and persistence. main.py owns the menu loop and delegates everything else. save_entry calls os.makedirs(..., exist_ok=True) so data/ is created on demand, then appends a timestamped record and rewrites the JSON file.

Section 14

Deep Dive: why raise instead of return?

Returning "Error: ..." mixes results with errors, forcing every caller to string-check. Raising an exception separates the happy path from failure handling and lets errors bubble up. The Advanced variant builds a custom CalculatorError for exactly this reason.
